# Agent命令使用示例

## 🎯 完整对话示例

### 示例1：使用产品经理Agent

```markdown
用户：/pm
或
用户：/agent:pm

Claude：
👋 你好！我是你的产品经理助手。

我的专长包括：
📋 需求分析与用户研究
📝 PRD文档和用户故事编写
🎯 功能规划与优先级排序
📊 竞品分析与市场调研
🔄 MVP规划与版本迭代

为了更好地帮助你规划产品，我需要了解以下信息：

1️⃣ **产品愿景**：你想要解决什么问题？
2️⃣ **目标用户**：谁会使用这个产品？
3️⃣ **核心价值**：产品能为用户带来什么价值？
4️⃣ **成功指标**：如何衡量产品成功？
5️⃣ **约束条件**：有哪些时间、资源或技术限制？

请先简单介绍你的产品想法，我会通过针对性的问题帮你完善需求。

---

用户：我想做一个在线教育平台，主要面向K12学生

Claude：
很好！在线教育是一个很有潜力的领域。让我深入了解一下你的想法：

📚 **关于K12在线教育平台**

首先，我想了解几个关键问题：

**1. 细分定位**
- 覆盖K12全阶段（小学、初中、高中）还是专注某个阶段？
- 主打哪些学科？全科还是特定科目（如数学、英语）？
- 是课程辅导、作业辅助还是素质教育？

**2. 教学模式**
- 录播课程、直播授课还是两者结合？
- 1对1、小班课还是大班课？
- 是否需要师生互动功能？

**3. 用户角色**
- 除了学生，家长是否也是重要用户？
- 老师端功能如何设计？
- 是否需要教务管理功能？

**4. 差异化竞争**
- 与作业帮、猿辅导等现有平台相比，你的独特价值是什么？
- 是否有特殊的教学方法或内容优势？

**5. 商业模式初步考虑**
- 按课程收费还是会员订阅制？
- 是否考虑免费增值模式？

请先回答这些问题，我会帮你整理成完整的产品规划。
```

### 示例2：使用UI设计师Agent

```markdown
用户：/ui
或
用户：/agent:ui

Claude：
🎨 你好！我是你的UI/UX设计师助手。

我的设计能力包括：
🖼️ 界面设计与视觉创意
🎯 用户体验优化
🎭 交互设计与动效
📐 设计系统构建
♿ 无障碍设计

为了创建最适合的设计方案，我想了解：

1️⃣ **设计风格**：你喜欢什么样的视觉风格？（现代、简约、活泼、专业）
2️⃣ **品牌调性**：有现有的品牌规范吗？主色调偏好？
3️⃣ **用户群体**：主要用户的年龄段和使用习惯？
4️⃣ **参考案例**：有喜欢的产品设计参考吗？
5️⃣ **设备适配**：主要在什么设备上使用？（手机、平板、桌面）

请描述你的设计需求，我会通过专业的设计思维帮你打造优秀的用户界面。

---

用户：基于刚才的在线教育平台，设计学生端的课程列表页面

Claude：
基于K12在线教育平台的定位，我来设计学生端课程列表页面：

🎨 **设计方案**

### 设计理念
考虑到K12学生的认知特点，设计需要：
- 活泼友好，避免过于严肃
- 信息层级清晰，易于理解
- 色彩丰富但不杂乱
- 加入游戏化元素增加趣味性

### 视觉风格
- **主色调**：明亮的蓝色（#4A90E2）- 代表知识和信任
- **辅助色**：
  - 橙色（#FFA500）- 活力和创造力
  - 绿色（#52C41A）- 成长和进步
- **风格定位**：卡通插画风 + 扁平化设计

### 页面布局

#### 顶部导航栏
- Logo + 搜索框
- 学科分类标签（可横向滑动）
- 个人中心入口

#### 筛选区域
- 年级选择（小学/初中/高中）
- 科目筛选（语数英理化生等）
- 难度等级（基础/提高/竞赛）
- 授课方式（录播/直播/1对1）

#### 课程卡片设计
每个课程以卡片形式展示：
- 课程封面图（带有吸引力的插画）
- 课程标题和简介
- 老师头像和姓名
- 评分星级（5星制）
- 学习人数
- 价格标签（或"免费"标识）
- 进度条（如果已开始学习）

#### 交互细节
- 卡片hover效果：轻微放大 + 阴影加深
- 点击进入课程详情
- 收藏按钮（爱心图标）
- 快速预览功能（播放按钮）

需要我详细设计某个部分或输出设计规范文档吗？
```

## 💡 使用技巧

### 1. 命令别名
- `/pm` = `/agent:pm` = 产品经理
- `/ui` = `/agent:ui` = `/designer` = UI设计师
- `/fe` = `/agent:fe` = `/frontend` = 前端工程师
- `/be` = `/agent:be` = `/backend` = 后端工程师

### 2. 连续对话
Agent会保持角色上下文，你可以连续对话：
```
/pm
[PM对话...]
继续细化需求...
[PM继续回应...]
```

### 3. 切换Agent
直接使用新命令切换：
```
/pm 分析需求
[PM回应...]
/ui 基于需求设计界面
[UI回应...]
```

### 4. 参数使用
```
/fe --framework react --lang ts
/be --lang python --db postgres
/team --mvp --agents pm,ui,fe
```
